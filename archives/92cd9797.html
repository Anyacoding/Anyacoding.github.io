使用MySQL

MySQL 是一种数据库管理系统，即它是一种数据库软件。MySQL 已经存在很久了，并在世界范围内得到了广泛的安装和使用。那么它有什么优点吸引了如此多的从业者和公司使用它呢？

以下列出一些原因：

  • 成本低 —— MySQL 是开源的，我们可以免费地使用它，甚至更改它的源码。
  • 性能高 —— MySQL 执行起来很快。
  • 可信赖 —— 有非常多的公司和站点都使用 MySQL 来处理自己的重要数据。
  • 上手快 —— MySQL 十分容易安装，并且学习成本低。

作为入门数据库来说，MySQL 是十分不错的选择。

检索单个列

SQL和大小写
请注意，SQL 语句不区分大小写，因此 SELECT 与 select 是相同的。然而许多业内人士喜欢对所有 SQL 关键字使用大写，而对所有列和表名使用小写，这样做有利于代码的阅读和调试。

不过需要意识到的是，虽然 SQL 语句不区分大小写，有些标识符却可能是区分大小写的，例如数据库名或者表名，这取决与 MySQL 的版本。

一种好的做法是坚持大小写惯例，并保持始终。

使用空格
SQL 语句可以在一行上给出，也可以分成许多行，当语句很长时，将语句分成多行有利于阅读和调试。

使用逗号

一般来说，除非你确实需要表中的每个列，否则最好不要使用 * 通配符。虽然你是省事了，但检索不需要的列会浪费性能。

检索不同的行

DISTINCT语句
DISTINCT 告诉 MySQL 只返回 ID 不同的每一行，因此可以做到检索不同行的作用。如果使用 DISTINCT 关键字，他必须直接放在所有列名的前面。

DISTINCT不能部分使用
DISTINCT 关键字应用于所有列而不仅仅是它的前置列。如果给出 DISTINCT id, name，除非指定的两个列都不同，否则所有行都会被检索出来。

OR操作符

计算次序

和大多数编程语言一样，在 SQL 语句中 AND 的优先级比 OR 要高。当两者组合使用时，为了保证结果如你所期望的给出，解决方法是使用圆括号。是的，就像你一直以来做的那样。

使用圆括号
不要过分依赖默认计算次序，即使它确实能得出你想要的结果。使用圆括号能够帮助你也帮助阅读该代码的人消除歧义。

IN操作符

虽然初看这个命令你可能会把它误以为类似于 Python 中的 range-for，但其实不是那么回事。圆括号中的是由逗号分隔的条件清单，他们每个都可以进行匹配。换句话说，IN 操作符其实就是 OR 操作符的高级版。


NOT操作符

MySQL中的NOT
MySQL 仅支持对 IN、BETWEEN 和 EXISTS 子句取反，这和其他的数据库管理系统支持对所有条件取反有些不同。

用通配符进行过滤

LIKE操作符

前面介绍的操作符都是针对已知值进行过滤的。当我们需要根据未知值进行过滤时，通配符就很用了。

通配符
用来匹配值的一部分特殊字符。

搜索模式
由字面值、通配符或两者组合构成的搜索条件。

百分号通配符

最常使用的通配符是百分号（%），在搜索中，% 表示 任何字符 出现 任意次数 。通配符可以在搜索模式的任意位置使用，并且可以使用多个通配符。



注意NULL
虽然 % 通配符似乎可以匹配任何东西，但有一个例外，那就是 NULL。具有 NULL 值的行将不会被匹配。

下划线通配符

另一个有用的通配符是下划线（_）。下划线的用途和 % 一样，但是下划线 只能匹配单个字符而不是多个字符 。

使用技巧

MySQL 的通配符很好用。但好用都是有代价的：通配符的处理速度一般会更慢。为了更好使用通配符，这里给出一些技巧建议：

  • 不要过度使用通配符，如果你能通过其他方式达到同样的目的，请那样做。
  • 在确实需要使用通配符时，除非万不得已，不要把他们用在搜索模式的开始处，这样搜索起来是最慢的。
  • 注意通配符的位置，放错地方很有可能返回预期外结果。

使用MySQL正则表达式

基本字符匹配

LIKE与REGEXP
在 LIKE 与 REGEXP 之间有一个重要的差别。如之前所说，LIKE 匹配整个列，如果被匹配的文本在列值中出现，LIKE 将不能完整匹配它，因此相应的行也不会返回。而 REGEXP 在列值内进行匹配，如果被匹配的文本在列值中出现，REGEXP 将会找到它，相应的行会被返回，这是一个非常重要的差别。

那么 REGEXP 能不能起到匹配整个列值的作用呢，答案是可以的，用后文介绍的定位符即可。

匹配不区分大小写
MySQL 中的正则表达式匹配不区分大小，为区分大小写，可以使用关键字 BINARY，例如 WHERE name REGEXP BINARY 'ONG';

进行OR匹配

两个以上的OR条件
可以给出两个以上的 OR 条件，例如，'1000|2000|3000' 将匹配 1000 或 2000 或 3000。

匹配几个字符之一

事实上，正则表达式 [xyz]ong 为 [x|y|z]ong 的缩写，两者是等价的。但是我们需要用 [] 来定义查找什么。我们直接来看看不加 [] 会发生什么：



我们可以看到，除非把字符 | 括在一个集合中，否则将应用于整个串。

字符集合也可以取反
在集合的开始处加一个 ^ 即可，[^xyz] 会匹配除这些字符以外的东西。

匹配范围

[1-5] 定义了一个范围，这个表达式将匹配 1 到 5，这也是一种简写形式。

匹配特殊字符

正则表达式语言由具有特定含义的特殊字符构成。我们已经看到 . [] | - 等特殊字符了，那么我们要如何匹配这些字符而不产生歧义呢？



我们只需要在特殊字符前面加上 \\ 即可，这种处理就是所谓的转义。

匹配 \
为了匹配反斜杠字符本身，需要使用 \\\。

\ 还是 \ \
你可能在别的正则表达式实现中看到用单个反斜杠即可完成转义。但 MySQL 要求两个反斜杠，其中 MySQL 自己解释一个，正则表示式库自己解释一个。这点跟 C++ 是一样的。

匹配字符类

为了方便工作，我们可以使用预定义的字符集：

类 说明
[:alnum:] 任意字母和数字（同[a-zA-Z0-9]）
[:alpha:] 任意字母（同[a-zA-Z]）
[:blank:] 空格和制表（同[\\t]）
[:cntrl:] ASCII 控制字符
[:digit:] 任意数字（同[0-9]）
[:print:] 任意可打印字符
[:graph:] 与 [:print:] 相同，但不包含空格
[:lower:] 任意小写字母
[:upper:] 任意大写字母
[:punct:] 既不在 [:alnum:] 又不在 [:cntrl:] 中的任意字符
[:space:] 包含空格在内的任意空白字符（同[\\f\\n\\r\\t\\v]）
[:xdight:] 任意十六进制数（同[a-fA-F0-9]）

匹配多个实例

目前为止使用的所有正则表达式都视图匹配单次出现，当我们想要匹配多次出现时，可以借助正则表达式重复元字符来完成：

元字符 说明
* 0个或多个匹配
+ 1个或多个匹配，等价于{1,}
? 0个或1个匹配，等价于{0,1}
{n} 指定数目匹配
{n,} 不少于指定数目的匹配
{n,m} 匹配数目的范围（m不超过255）

定位符

元字符 说明
^ 文本的开始
$ 文本的结尾
[[:<:]] 词的开始
[[:>:]] 词的结尾

^的双重用途
^ 有两种用途，在集合的开头使用用来表示集合的反: [^123]。否则用来指示串的开始。

简单的正则表达式测试
可以在不使用数据库的情况下用 SELECT 来测试正则表达式：SELECT 'hello` REGEXP '[0-9]'
